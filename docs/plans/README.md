# Plans

本目录用于存放仍有实现价值的方案文档，不承担“当前真相”职责。

## 放什么

- 前端大版本重构方案
- feature 设计方案
- 核心流程改版方案
- 数据结构或接口边界的重要调整方案
- 重大 prompt / workflow 调整方案

## 不放什么

- 日常工作记录
- 临时排障过程
- 一次性修复说明
- 已失效且无人再依赖的旧草案

## 命名建议

- 优先使用清晰、语义化的英文 kebab-case 文件名
- 如果确实需要体现时间，再在文件名前加日期
- 文件名优先表达主题，不优先表达会议来源或聊天来源

## 推荐结构

- `frontend-v3-refactor-plan.md`
- `translation-workspace-plan.md`
- `indexing-backend-refactor-plan.md`

## 使用原则

- 方案文档记录取舍、边界和阶段计划，不记录运行态事实。
- 已被更新方案替代的旧文档，应删除或明确退役，不长期堆积。
- 当某项方案已稳定落地，其长期有效事实应被提炼进 `docs/CURRENT_STATE.md` 或其他稳定文档，而不是继续停留在方案文档中。
