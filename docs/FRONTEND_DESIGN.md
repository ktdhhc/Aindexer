# FRONTEND_DESIGN

> 稳定设计约束文档 | 适用于 V3 及后续版本的首版前端实现  \n> 基于「深空灰」方案 —— 深色沉浸 · 代码感 · 微光点缀

---

## 1. 文档目的

- 让任何后续开发者一眼知道“界面应该长什么样”
- 守住首版必须保持的视觉与交互边界，防止风格漂移
- 明确“什么复杂度不能引入”，避免过度设计
- 作为编码前即可引用的单一事实来源，不依赖口头约定

---

## 2. 前端定位

- **产品类型**：本地桌面式工作台，非公开网站
- **核心目标**：帮助用户快速完成“上传-索引-搜索-预览-翻译”闭环
- **使用场景**：长时间盯屏、夜间使用、学术文献阅读、高频操作
- **气质关键词**：沉浸、代码感、低饱和、无噪音、暗色优先

---

## 3. 总体设计原则

1. **深色优先** —— 默认暗色主题，白天可切换浅色，但设计以深色为基准
2. **内容即界面** —— 让文献、索引、翻译结果成为视觉主角，UI 退到背景
3. **零装饰** —— 无渐变背景、无大面积插图、无夸张圆角、无阴影堆叠
4. **操作优先** —— 任何页面首屏必须让用户知道“下一步可点什么”
5. **低文案** —— 能用标签就不用句子，能省略就省略，教学文案放到帮助中心
6. **一致性 > 创新** —— 与代码编辑器视觉语言保持一致，降低认知负荷

---

## 4. 视觉基调

| 关键词 | 解释 |
|--------|------|
| 深空灰 | 背景三层灰度 (#121212 → #1E1E1E → #2A2A2A)，永不使用纯黑 |
| 微光蓝 | 主强调色 #4F8CFF，仅用于可操作元素（链接、按钮、进度、选中） |
| 低饱和 | 辅助色饱和度 ≤ 40%，避免抢夺注意力 |
| 等宽数字 | 所有指标、时间戳、页码、进度条使用等宽字体，确保对齐 |
| 1 px 法则 | 分割线、描边、进度条统一 1 px，绝不使用 2 px 或阴影模拟 |

---

## 5. 颜色规范

```scss
// 背景层级 —— 离用户越近越亮
$bg-deep:     #121212;   // 最底层（全局背景）
$bg-surface:  #1E1E1E;   // 卡片/面板底层
$bg-float:    #2A2A2A;   // 悬停/弹层/输入框

// 文字层级 —— 对比度 ≥ 4.5 : 1
$text-primary:   #E4E4E4;   // 主文本
$text-secondary: #999999;   // 副文本
$text-disabled:  #666666;   // 禁用

// 状态色
$accent:      #4F8CFF;   // 主强调（可操作）
$success:     #66D9A6;   // 成功/完成
$warning:     #E5B567;   // 警告/重试
$error:       #FF6A6A;   // 错误/失败

// 分割线
$border: rgba(255,255,255,0.08);
```

**使用规则**
- 背景层级只能按 `$bg-deep → $bg-surface → $bg-float` 顺序上浮，禁止跳级
- 强调色仅用于“可交互”元素，禁止用于纯装饰或静态标签
- 错误色出现时，必须同时提供「可执行」的修复动作，不允许仅展示错误文案

## 6. 字体与排版

| 用途 | 字号 | 行高 | 字重 | 字体 |
|------|------|------|------|------|
| 一级标题 | 28 px | 36 px | 600 | Inter / 思源黑体 |
| 二级标题 | 20 px | 28 px | 600 | 同上 |
| 正文 | 14 px | 20 px | 400 | 同上 |
| 小字/标注 | 12 px | 16 px | 400 | 同上 |
| 数字指标 | 14 px | 20 px | 500 | JetBrains Mono / SF Mono |

**规则**
- 全篇禁止使用 18 px、22 px 等“中间值”，保持 4 px 步进节奏
- 段落文本最大宽度 640 px，超过必须分栏
- 任何文本禁止压到 14 px 以下（代码块除外）

---

## 7. 间距与圆角

| 元素 | 圆角 | 内边距 | 外边距 |
|------|------|--------|--------|
| 大卡片 | 8 px | 24 px | 24 px |
| 小卡片/按钮 | 6 px | 12 px | 12 px |
| 输入框 | 4 px | 8 px 水平 10 px 垂直 | 0（由父容器统一负责） |
| 分割线 | 0 px | — | 16 px 上下 |

**规则**
- 所有间距遵循 4 px 基数网格，禁止出现 5、7、9 等奇数
- 同一容器内卡片圆角必须一致，不允许“大卡片 8 px 里套小卡片 12 px”

---

## 8. 页面布局规范

### 8.1 通用骨架

```
┌-------------------------------┐
| TopBar (48 px)  导航+全局操作 |
├-------------------------------┤
|                               |
|  WorkArea  （自适应剩余高度） |
|                               |
└-------------------------------┘
```

- TopBar 永远固定，不随内容滚动
- WorkArea 允许再分左右：左侧主内容 / 右侧侧边栏（翻译页、预览页）
- 侧边栏宽度 320 px 或 400 px 两种规格，禁止任意值

### 8.2 内容区规则

- 首屏必须露出「下一步可操作按钮」——不允许用户先滚动才能看到动作
- 列表页空状态仅允许一句话 + 一个主按钮，禁止大段说明
- 任何表单区域宽度 ≤ 560 px，超过必须分栏或分组

---

## 9. 组件最小集合

首版仅允许以下 9 类组件存在，禁止提前扩展：

1. Button（主/次/线框/图标）
2. Input + Textarea（含错误态）
3. Select（单选）
4. Checkbox & Radio
5. Table（纯展示，不含分页、排序、筛选）
6. Card（大/小两种圆角）
7. Divider（横线）
8. Loading（线性 2 px 进度条 + 文字）
9. Modal（居中，最大宽 640 px，带遮罩）

**规则**
- 所有组件必须提供暗色样式，浅色可延后
- 禁止引入第三方“大而全”组件库（如 AntD、Mantine、Chakra）
- 确实需要复杂交互（下拉树、穿梭框）时，先简化需求，再考虑自研

---

## 10. 关键页面规范

### 10.1 控制台列表页

- 顶部：搜索框（最大宽 400 px）+ 排序下拉 + 主按钮「上传」
- 列表：每行仅展示 4 个字段（标题、作者、状态、更新时间）+ 操作按钮组（查看/编辑/导出）
- 底栏：极简分页（上一页/下一页），禁止一次性跳 10 页

### 10.2 翻译工作区

- 左侧 PDF 预览区占比 60 %，右侧翻译侧边栏 40 %
- 顶部工具栏固定：文档名、搜索、页码、关闭
- 选区翻译按钮仅在划词后出现，位置跟随光标上方 8 px
- 翻译结果卡片内部禁止再嵌套卡片，仅用 1 px 描边区分原文/译文

### 10.3 配置页（Provider / 字段）

- 一屏最多一个主按钮（保存），其余操作放次按钮
- 表单分组标题使用 20 px 二级标题，分组间距 32 px
- 输入框右侧必须预留 48 px 区域给「测试/重置」图标按钮，避免换行

---

## 11. 状态展示规范

| 状态 | 视觉 | 文案 | 动作 |
|------|------|------|------|
| 加载 | 2 px 线性进度条 | 不显示文案，仅用进度条 | 不可操作区域置灰 |
| 空数据 | 大号图标 + 一句话 | 不超过 14 个汉字 | 提供唯一主按钮 |
| 错误 | 红色左侧条 + 一行错误 | 给出可执行建议 | 提供「重试」或「去修复」 |
| 成功 | 绿色左侧条，2 s 消失 | 不超过 10 个汉字 | 无动作，自动消失 |

**规则**
- 禁止用弹窗报告成功，仅用顶部横幅
- 错误文案必须包含“原因 + 下一步动作”，不允许只写“出错了”

---

## 12. 表格 / 表单 / Evidence / 差异展示规则

### 12.1 表格

- 表头字重 500，文字与背景对比度 ≥ 4.5 : 1
- 行高 40 px，斑马纹禁用，仅用 1 px 横线分割
- 操作列放置最右侧，宽度 120 px，最多两个按钮（图标 + 文字）

### 12.2 表单

- 标签右对齐，宽度统一 120 px，冒号省略
- 必填仅红星 *，位置在标签左侧
- 错误提示出现在输入框下方 4 px，红色 12 px 文字

### 12.3 Evidence（文献元数据展示）

- 采用“键: 值”两栏，键用 $text-secondary，值用 $text-primary
- 折行使用悬挂缩进，值左侧对齐到键右侧 8 px
- 超过 3 行自动折叠，提供「展开」链接

### 12.4 差异展示（Markdown 编辑对比）

- 新增行背景 #0A2D1B（绿色 10 % 透明度）
- 删除行背景 #2D0A0A（红色 10 % 透明度）
- 禁止用红绿文字，仅用背景区分，避免色盲障碍

---

## 13. 动效与响应式约束

### 13.1 动效时长

| 场景 | 时长 | 曲线 |
|------|------|------|
| 路由切换 | 200 ms | ease-out |
| 弹窗出现 | 150 ms | ease-out |
| hover 反馈 | 100 ms | linear |
| 进度条 | 线性 | linear |

**禁止**
- 弹性、回弹、3D 翻转
- 超过 300 ms 的任何动画
- 同时出现 3 个以上动画元素

### 13.2 响应式断点

- ≥ 1280 px：左侧导航固定 220 px，内容区自适应
- 960–1279 px：导航收起为图标抽屉，内容区占满
- < 960 px：转为底部标签栏，隐藏侧边栏

**规则**
- 仅支持两种布局：侧边导航 / 底部标签，禁止中间状态
- 移动端优先保证“上传、搜索、翻译”三大动作首屏可见

---

## 14. 开发减负规则

1. 禁止提前封装“未来可能复用”的抽象组件<br>→ 至少出现 3 次真实使用场景才提取
2. 禁止引入图标库全量包<br>→ 仅用 Heroicons 或 Tabler 中已用到的 48 枚图标，其余用字体符号
3. 禁止自定义 CSS 动画<br>→ 全部用 tailwind 内置 animate-*，够用即可
4. 禁止在组件内部写媒体查询<br>→ 统一用 Tailwind 断点 class，保持原子化
5. 禁止为低概率场景写分支逻辑<br>→ 先让 95 % 主流程极致稳定，再考虑边缘情况

---

## 15. 验收标准

✅ **视觉一致性**<br>
任意两个页面截图并排，字体、圆角、间距、强调色肉眼不可察觉差异

✅ **交互一致性**<br>
同一动作（保存、删除、上传）在不同页面触发后，反馈样式与时长相差 < 50 ms

✅ **性能底线**<br>
首屏渲染 ≤ 200 ms（本地 dev 构建），路由切换 ≤ 100 ms，列表滚动 60 FPS 无掉帧

✅ **可访问性**<br>
所有表单元素可通过键盘完整走通；对比度均 ≥ 4.5 : 1；屏幕阅读器可读出状态变化

✅ **代码边界**<br>
新增组件必须落在“组件最小集合”列表内；任何 PR 引入新第三方 UI 库需经架构评审
