# V3.5 Editorial Lab Frontend Design

> 稳定设计约束文档 | 适用于 Aindexer V3.5 正式产品前端  
> 方向：Editorial Lab —— 学术编辑台 · 暖色纸张感 · 阅读与修订优先

---

## 1. 文档目的

- 定义 V3.5 Editorial Lab 方案的视觉风格、页面结构和交互边界。
- 将 Aindexer 从普通工具界面提升为带有学术出版、批注和编辑气质的本地工作台。
- 为后续 React V3.5 迁移提供可执行的设计约束。
- 本文档描述“应该长什么样”，不描述当前实现细节。

---

## 2. 前端定位

- **产品类型**：本地桌面式文献编辑、索引修订、翻译校对与知识整理工作台。
- **核心目标**：让用户围绕文献内容完成检索、阅读、修订、翻译和导出。
- **使用场景**：长时间阅读、人工校对、批注文献、整理翻译片段、维护结构化索引。
- **气质关键词**：暖白纸张、编辑台、学术出版、沉静、精致、内容主导。
- **默认主题**：暖浅色主主题，夜间深棕/墨色主题作为后续辅助。

---

## 3. 总体设计原则

1. **阅读优先** —— 预览、正文和翻译结果拥有最高视觉权重。
2. **编辑优先** —— 修订、标注、差异和导出流程必须顺手。
3. **低噪声** —— 不使用 dashboard 式大量指标卡，不用强科技感装饰。
4. **纸张层级** —— 用纸张色、墨色、细线和排版建立层次。
5. **操作收敛** —— 高频动作留在上下文附近，低频配置移到设置页。
6. **学术可信** —— 字体、行距、引用、元数据展示要像正式编辑工具。
7. **克制品牌感** —— 产品应有识别度，但不能压过文献内容。

---

## 4. 视觉基调

| 关键词 | 解释 |
|--------|------|
| Editorial Lab | 文献编辑室，而不是控制台后台 |
| 暖白纸张 | 背景接近纸张与档案盒，长时间阅读更舒适 |
| 墨色文字 | 主文本接近印刷墨色，避免纯黑压迫 |
| 铜橙强调 | 作为保存、选中、修订、批注的唯一强调色 |
| 内容画布 | 预览区像可编辑稿纸，左右工具退到次级 |

---

## 5. 颜色规范

```scss
// 背景层级
$bg-page:      #F3EEE6;   // 全局暖纸背景
$bg-surface:   #FFFDF8;   // 主文档纸面
$bg-subtle:    #E9E0D3;   // 侧栏、列表 hover、分隔区
$bg-elevated:  #FFFCF3;   // 弹层、输入框、浮层

// 文本层级
$text-primary:   #241C16; // 主文字，接近墨色
$text-secondary: #67584B; // 次级元数据
$text-muted:     #9A8B7A; // 弱提示
$text-inverse:   #FFFDF8;

// 品牌与动作
$accent:       #A65F2B;   // 铜橙强调
$accent-hover: #87471E;
$accent-soft:  #F3DFCB;

// 状态色
$success: #2F7D55;
$warning: #A65F2B;
$error:   #B23A2E;
$info:    #3A6785;

// 边界
$border-strong: rgba(36, 28, 22, 0.18);
$border:        rgba(36, 28, 22, 0.12);
$border-soft:   rgba(36, 28, 22, 0.07);
```

**使用规则**

- 铜橙只用于主要动作、选中态、批注标记和关键状态。
- 页面不使用冷蓝科技感渐变，不使用荧光色。
- 文献预览必须使用 `$bg-surface`，保持稿纸感。
- 状态色保持低饱和，不抢正文阅读注意力。
- 错误色必须配合修复动作，不只显示错误文案。

## 6. 字体与排版

| 用途 | 字号 | 行高 | 字重 | 字体 |
|------|------|------|------|------|
| 产品标题 | 26 px | 34 px | 600 | Source Serif 4 / Noto Serif SC |
| 页面标题 | 22 px | 30 px | 600 | Source Serif 4 / Noto Serif SC |
| 区块标题 | 15 px | 22 px | 650 | IBM Plex Sans / Noto Sans SC |
| 正文 UI | 14 px | 20 px | 400 | IBM Plex Sans / Noto Sans SC |
| 辅助文本 | 12 px | 16 px | 450 | IBM Plex Sans / Noto Sans SC |
| 数字/时间/ID | 13 px | 18 px | 500 | JetBrains Mono / SF Mono |
| 文献预览 | 16 px | 28 px | 400 | Source Serif 4 / Noto Serif SC |

**规则**

- 标题可带轻微编辑出版气质，但 UI 控件仍使用无衬线字体。
- 文献正文优先保证行距和段落节奏，不能被控件挤压。
- 主界面不要使用过多大标题，避免像杂志首页。
- 元数据和操作文案保持短，避免破坏编辑台的安静感。

---

## 7. 间距与圆角

| 元素 | 圆角 | 内边距 | 说明 |
|------|------|--------|------|
| 文档画布 | 18 px | 28-40 px | 重点区域，类似稿纸 |
| 工具面板 | 14 px | 14-18 px | 轻边框、低阴影 |
| 按钮 | 999 px 或 10 px | 8-14 px | 主按钮可胶囊形 |
| 输入框 | 10 px | 0 12 px | 高度 38/42 px |
| 状态标签 | 999 px | 4 8 px | 保持紧凑 |

**规则**

- 间距使用 4 px 基数：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40。
- 预览区内边距大于列表区，强化阅读空间。
- 不使用玻璃拟态和强阴影，最多使用纸张投影。
- 工具区不堆叠厚卡片，避免像普通后台。

## 8. 页面布局规范

### 8.1 通用骨架

```text
┌────────────────────────────────────────────┐
│ TopBar 60px：产品名 / Workspace / 状态     │
├──────────────┬─────────────────────────────┤
│ Library Rail │ Editorial Workspace         │
│ 72px         │ 文献列表 / 文档画布 / 批注栏 │
└──────────────┴─────────────────────────────┘
```

- TopBar 固定高度 60 px，视觉上更像编辑软件而非管理后台。
- 左侧 Library Rail 提供主功能入口：文库、索引、翻译、配置。
- 工作区以“文献列表 + 文档画布 + 批注/助手栏”为核心。

### 8.2 工作台布局

```text
┌ Library Banner: 当前档案 / 关键状态 / 主行动 ┐
┌ Toolbar: Upload / Index / Template / Provider ┐
├────────────┬──────────────────────┬───────────┤
│ Library    │ Document Canvas      │ Notes     │
│ 300-360px  │ fluid                │ 300-340px │
└────────────┴──────────────────────┴───────────┘
```

- 文库页内容顶部允许加入一个非卡片式横幅，作为页面空间锚点。
- 横幅优先展示当前 Workspace 的关键词或词云，并辅以当前档案、待处理状态、最近保存或索引状态，以及一个主行动。
- 横幅不使用卡片边框和厚背景，不承载说明文案，不堆叠指标卡。
- 文档画布是主视觉中心，背景为纸张色。
- Library 列表负责搜索、筛选、排序和文档切换。
- Notes 栏承载 Chat、索引字段、翻译片段和审核状态。
- 上传与索引动作位于顶部工具条，不占用正文画布。

### 8.3 配置页布局

- 配置页应比工作台更简洁，减少纸张装饰。
- 表单最大宽度 720 px，分组使用细线和标题分隔。
- 删除、重置等危险操作固定在区块底部，不与保存并列抢权重。

### 8.4 翻译页布局

- 原文/PDF 画布和译文侧栏形成双栏校对台。
- 译文结果使用段落块，不使用小卡片堆叠。
- 选区、批注、历史记录应沿用铜橙标记系统。

---

## 9. 组件最小集合

首版仅允许以下组件：

1. Button（primary / secondary / text / danger / icon）
2. Input / Textarea
3. Select / Combobox
4. Checkbox / Radio / Switch
5. DocumentList
6. PaperPanel（文档画布）
7. ToolPanel（工具面板）
8. Divider
9. StatusPill / ProgressBar
10. Modal / SideSheet
11. Toast / InlineAlert
12. Tabs / SegmentedControl

**规则**

- 不引入大而全组件库。
- PaperPanel 只用于正文、预览、编辑、翻译结果，不泛滥到所有区域。
- ToolPanel 内禁止再嵌套多层卡片。
- 空状态最多一句话和一个动作。

---

## 10. 关键页面规范

### 10.1 工作台首页

- 页面内容顶部显示文库横幅，突出当前 Workspace、文献总量、待审核和最近任务。
- 顶部工具条只保留上传、Provider、字段模板、生成索引，低频设置不进入首屏。
- 左侧 Library 默认展示当前 Workspace 的文献列表。
- 文献行主标题为 `display_name`，次信息为作者、年份、状态。
- 中间 Document Canvas 默认展示索引预览或选中文档的空状态。
- 右侧 Notes 显示索引状态、字段摘要、Chat 和待审核动作。
- 编辑动作应靠近文档画布，而不是隐藏在列表行。

### 10.2 配置页

- Provider 设置采用清晰表单，不加说明卡片。
- 字段模板像“编辑规则集”，强调排序、启用、必填、字段类型。
- Workspace 管理像“项目档案柜”，支持新建、重命名、删除。
- 保存按钮固定在当前表单底部或顶部右侧。

### 10.3 翻译工作区

- 原文画布保留足够行距和页边距。
- 译文侧栏按“当前选区 / 替代表达 / 历史”组织。
- 搜索命中和选区高亮使用铜橙透明底。
- 翻译结果可复制、替换、加入笔记。

### 10.4 Chat 页面

- 长会话采用“左侧会话列表 + 中间消息 + 右侧引用”结构。
- 来源引用以文献条目和段落摘录展示，不做花哨气泡。
- 消息内容优先可读性，避免聊天软件式过度装饰。

## 11. 状态展示规范

| 状态 | 视觉 | 文案 | 动作 |
|------|------|------|------|
| 加载 | 顶部细进度条 / 行内骨架 | 不超过一句 | 禁用相关动作 |
| 空数据 | 纸张画布中心一句话 | “暂无文档” | 上传 |
| 处理中 | 铜橙进度条 + 状态 pill | 阶段名 | 取消 |
| 需审核 | 铜橙左边条 | 需审核 | 打开编辑 |
| 错误 | 红色左边条 | 原因 + 下一步 | 重试 / 配置 |
| 成功 | 顶部轻 toast | 不超过 10 个字 | 无 |

**规则**

- 成功不弹窗。
- 删除、重置、取消必须二次确认。
- 审核态与错误态要明显区分，避免把可修订内容当成失败。
- 翻译失败应保留原文选区，不丢失上下文。

---

## 12. 表格 / 列表 / Evidence / 差异展示规则

### 12.1 文献列表

- 行高 72-92 px。
- 选中态使用左侧铜橙线和暖色背景。
- 作者、年份、关键词作为次级信息，避免压过标题。

### 12.2 表格

- 表格只用于配置和字段模板，不用于主文档预览。
- 表头 12 px，字重 650，背景为 `$bg-subtle`。
- 行高 42 px，分隔线 1 px。

### 12.3 Evidence

- Evidence 像脚注或索引卡片，键宽 96 px。
- 引用来源必须可跳转到文档画布。
- 过长字段折叠，展开动作使用 text button。

### 12.4 Markdown 差异

- 新增背景使用 `rgba(47,125,85,.12)`。
- 删除背景使用 `rgba(178,58,46,.12)`。
- 批注背景使用 `rgba(166,95,43,.14)`。
- 不只依赖文字颜色区分差异。

## 13. 动效与响应式约束

| 场景 | 时长 | 曲线 |
|------|------|------|
| 路由切换 | 140 ms | ease-out |
| SideSheet / Modal | 180 ms | ease-out |
| Hover / Focus | 100 ms | linear |
| 文档切换 | 160 ms | ease-out |

**规则**

- 动效模拟编辑软件的轻微切换，不做弹跳、漂浮和 3D。
- 文档切换可以轻微淡入，但不能影响阅读定位。
- ≥1280 px 使用 Library + Canvas + Notes 三栏。
- 960-1279 px Notes 折叠为抽屉。
- <960 px 转单列，优先保留 Library、Canvas、上传入口。

---

## 14. 开发减负规则

1. 文档画布、工具面板、列表行是最重要的三个基础形态。
2. 同一 UI 模式出现 3 次后再提取共享组件。
3. 不把每个区块都做成 PaperPanel，避免样式膨胀。
4. 页面不直接拼接复杂请求参数，统一走 API client。
5. 服务端状态交给 TanStack Query，Zustand 只保存 UI 状态。
6. 帮助说明进入文档页，不进入主工作区。

---

## 15. 验收标准

✅ **产品感**  
首屏像正式的学术编辑工具，而不是 demo 管理后台。

✅ **阅读舒适**  
文献预览可长时间阅读，正文、元数据、操作不互相抢夺。

✅ **编辑效率**  
用户能快速找到修订、复制、导出、翻译和审核动作。

✅ **视觉一致性**  
所有页面共享暖纸背景、墨色文本、铜橙强调和细线边界。

✅ **可维护性**  
不允许工作台页继续形成超过 600 行的单页巨石组件。

✅ **可访问性**  
核心操作可键盘完成，对比度满足 WCAG AA。

---

> 本方案适合把 Aindexer 做成“学术编辑与阅读工作台”。如果优先目标是高密度操作和仪器感，应对比 Research OS 方案。
