# Research OS Frontend Design

> 稳定设计约束文档 | 适用于 Aindexer 正式产品前端候选方案  
> 方向：Research OS —— 浅色科研工作台 · 仪器面板感 · 高密度但低噪声

---

## 1. 文档目的

- 定义 Research OS 方案的视觉风格、页面结构和交互边界。
- 将产品从 demo 感的拼接界面推进为稳定、专业、可长期维护的本地研究工作台。
- 为后续 React V3 重构提供可直接执行的设计约束。
- 本文档描述“应该长什么样”，不描述当前实现细节。

---

## 2. 前端定位

- **产品类型**：本地桌面式文献索引、检索、修订与翻译工作台。
- **核心目标**：让用户高效完成“上传 -> 索引 -> 搜索 -> 预览 -> 修订 -> 导出/翻译”闭环。
- **使用场景**：长时间阅读、批量整理文献、低频配置、高频检索与预览。
- **气质关键词**：冷静、清晰、科研仪器、低装饰、高可信、信息密度高。
- **默认主题**：浅色主主题，深色模式作为后续辅助主题。

---

## 3. 总体设计原则

1. **内容优先** —— 文献、索引、预览和状态是视觉主角，UI 只提供结构和控制。
2. **操作优先** —— 首屏直接露出上传、索引、搜索、预览等核心动作。
3. **低文案** —— 主界面不承载说明书，文案只用于定位、状态、动作和错误。
4. **面板克制** —— 使用分隔线、留白和轻背景建立层级，少用厚重卡片。
5. **研究感而非后台感** —— 界面像本地研究工具，不像 SaaS 仪表盘和营销后台。
6. **状态可扫读** —— 任务、索引、失败、需审核必须一眼可见。
7. **长期维护优先** —— 组件少、样式 token 稳定、页面按工作流拆分。

---

## 4. 视觉基调

| 关键词 | 解释 |
|--------|------|
| Research OS | 类似科研操作系统的工作表面，稳定、冷静、工具化 |
| 浅色仪器面板 | 以浅灰、白、青绿强调构成可长期阅读的低噪声界面 |
| 弱卡片 | 区块主要靠边界、标题、留白和背景层级区分 |
| 数据可信感 | 数字、状态、时间、进度使用等宽字体并对齐 |
| 文献友好 | 预览正文使用更适合阅读的衬线字体或高可读正文字体 |

## 5. 颜色规范

```scss
// 背景层级
$bg-page:      #F4F6F5;   // 全局底色
$bg-surface:   #FFFFFF;   // 主工作区表面
$bg-subtle:    #EEF2F1;   // 次级区域、侧栏、列表 hover
$bg-elevated:  #F9FBFA;   // 弹层、浮层、输入框

// 文本层级
$text-primary:   #17211F; // 主要文本
$text-secondary: #5E6B67; // 次级文本
$text-muted:     #8A9692; // 弱提示、空状态
$text-inverse:   #FFFFFF;

// 品牌与动作
$accent:       #0F8F7F;   // 主强调：运行、保存、发送、选中
$accent-hover: #0B7468;
$accent-soft:  #DDF3EF;

// 状态色
$success: #12805C;
$warning: #A76614;
$error:   #B42318;
$info:    #2563A8;

// 边界
$border-strong: rgba(23, 33, 31, 0.16);
$border:        rgba(23, 33, 31, 0.10);
$border-soft:   rgba(23, 33, 31, 0.06);
```

**使用规则**

- 强调色只用于可操作元素、选中态和关键状态，不用于大面积装饰。
- 页面中不使用大面积渐变，不用霓虹色，不用高饱和紫色。
- 错误色出现时必须配合可执行动作，如“重试”“打开配置”“查看详情”。
- 预览正文保持高对比度，禁止低灰文本承载长内容。
- 同一个状态在所有页面使用同一个色值和文案。

## 6. 字体与排版

| 用途 | 字号 | 行高 | 字重 | 字体 |
|------|------|------|------|------|
| 产品标题 | 24 px | 32 px | 650 | IBM Plex Sans / Noto Sans SC |
| 页面标题 | 20 px | 28 px | 650 | IBM Plex Sans / Noto Sans SC |
| 区块标题 | 15 px | 22 px | 650 | IBM Plex Sans / Noto Sans SC |
| 正文 UI | 14 px | 20 px | 400 | IBM Plex Sans / Noto Sans SC |
| 辅助文本 | 12 px | 16 px | 450 | IBM Plex Sans / Noto Sans SC |
| 数字/时间/ID | 13 px | 18 px | 500 | JetBrains Mono / SF Mono |
| 文献预览 | 15 px | 25 px | 400 | Source Serif 4 / Noto Serif SC |

**规则**

- 主界面字号不低于 12 px，正文默认 14 px。
- 列表标题与预览标题不要过度放大，避免像营销页面。
- 文献正文和 UI 控件字体可以区分，但全局不超过三种字体角色。
- 数字、进度、年份、任务耗时、文件 ID 使用等宽字体。

## 7. 间距与圆角

| 元素 | 圆角 | 内边距 | 说明 |
|------|------|--------|------|
| Shell / 主工作区 | 0-20 px | 16-24 px | 只在需要桌面应用感时使用大圆角 |
| 面板 | 16 px | 16-20 px | 默认轻边框，不使用厚阴影 |
| 按钮 | 10 px | 8-14 px | 高度 36/40 px 两档 |
| 输入框 | 10 px | 0 12 px | 高度 38/42 px 两档 |
| 状态标签 | 999 px | 4 8 px | 紧凑、可扫读 |

**规则**

- 间距使用 4 px 基数：4 / 8 / 12 / 16 / 20 / 24 / 32。
- 主工作区不做卡片马赛克，优先用三栏布局和水平分隔。
- 不使用强投影；浮层最多使用 0 12px 28px rgba(23,33,31,.12)。

## 8. 页面布局规范

### 8.1 通用骨架

```text
┌────────────────────────────────────────────┐
│ TopBar 56px：产品名 / Workspace / 全局状态 │
├──────────────┬─────────────────────────────┤
│ Rail 64px    │ WorkArea                    │
│ 一级功能导航 │ 具体页面工作区               │
└──────────────┴─────────────────────────────┘
```

- TopBar 固定高度 56 px，承载产品识别、当前 Workspace、全局状态和少量全局动作。
- 左侧 Rail 固定 64 px，图标 + 短标签，避免宽侧栏占用工作区。
- WorkArea 根据页面再拆分为列表区、预览区、检查器/助手区。

### 8.2 工作台布局

```text
┌ Upload / Controls / Queue ┐
├────────────┬──────────────┬────────────┤
│ Search     │ Preview      │ Assistant  │
│ 320-380px  │ fluid        │ 320-360px  │
└────────────┴──────────────┴────────────┘
```

- 搜索列固定宽度，承担检索、排序和结果选择。
- 预览列为主视觉中心，承担 Markdown/渲染预览和编辑入口。
- Assistant/Inspector 为可折叠侧栏，承载 Chat、文档元数据、任务细节。
- 上传与任务控制区保持在首屏顶部，不能被折叠到不可见位置。

### 8.3 配置页布局

- 左侧为配置分组导航：Provider、字段模板、Workspace、系统偏好。
- 右侧为当前表单，最大宽度 760 px。
- 表单不做营销式说明，必要说明放字段 tooltip 或文档页。

### 8.4 翻译页布局

- 左侧 PDF/原文预览占 58%-64%。
- 右侧翻译侧栏占 36%-42%。
- 顶部文档工具条固定，页码、搜索、关闭、Provider 选择保持可达。

## 9. 组件最小集合

首版仅允许以下组件：

1. Button（primary / secondary / ghost / danger / icon）
2. Input / Textarea
3. Select / Combobox（仅 Provider、模板、Workspace 使用）
4. Checkbox / Radio / Switch
5. Table / DenseList
6. Panel（弱卡片，不做多层嵌套）
7. Divider
8. StatusPill / ProgressBar
9. Modal / Drawer
10. Toast / InlineAlert
11. Tabs / SegmentedControl

**规则**

- 不引入 AntD、Mantine、Chakra 等大组件库。
- Panel 内部禁止再堆多层卡片，除非子项本身是可点击列表行。
- 空状态只允许一句话和一个动作。
- 图标只用于提升识别，不用于填充装饰。

---

## 10. 关键页面规范

### 10.1 工作台首页

- 首屏显示上传入口、Provider、字段模板、批量索引、任务队列摘要。
- 搜索列默认展示当前 Workspace 的全部可检索文档。
- 列表行主标题为 `display_name`，次信息为年份、作者、状态、关键词。
- 点击列表行加载预览，不要求用户输入 doc_id。
- 预览区默认只读，编辑通过 Modal 或右侧 Inspector 完成。
- Chat 作为侧栏能力存在，不抢占预览主区域。

### 10.2 配置页

- Provider、字段模板、Workspace 分组清晰，但共享同一表单语言。
- 一屏只保留一个主动作，如“保存”。
- 测试连接、重置、删除等动作用 secondary/warning/danger 区分。
- API Key 明确显示“已保存/未配置”状态，不展示明文。

### 10.3 翻译工作区

- PDF 或文本预览是主视觉中心。
- 选区翻译按钮只在选中文本后出现。
- 翻译结果侧栏按“当前结果 / 历史 / 设置”组织。
- 翻译历史不使用厚卡片，使用时间线或分隔列表。

### 10.4 Chat 页面

- 独立 Chat 页用于长会话和调试，不替代工作台侧栏 Chat。
- 会话配置固定在顶部或左侧，消息列表占主区域。
- 引用来源必须可展开查看原文片段或跳转预览。

## 11. 状态展示规范

| 状态 | 视觉 | 文案 | 动作 |
|------|------|------|------|
| 加载 | 1 px 顶部进度条或行内骨架 | 不超过一句 | 禁用相关动作 |
| 空数据 | 一行浅灰文字 | “暂无文档” | 上传 / 新建 |
| 处理中 | 青色进度条 + 状态 pill | 阶段名 | 取消 |
| 需审核 | 琥珀色左边条 | 需审核 | 打开编辑 |
| 错误 | 红色左边条 | 原因 + 下一步 | 重试 / 配置 |
| 成功 | 轻量 toast | 不超过 10 个字 | 无 |

**规则**

- 成功不弹窗。
- 删除、重置、取消任务必须二次确认。
- 任务状态和文档状态不要混用文案。
- 错误信息应说明下一步，不写“未知错误”作为唯一文案。

---

## 12. 表格 / 列表 / Evidence / 差异展示规则

### 12.1 列表

- 行高 64-88 px，根据元数据密度调整。
- 选中态使用左侧 3 px accent 条 + 轻背景。
- 行级操作最多 3 个，默认在 hover 或选中时强化。

### 12.2 表格

- 表头 12 px、字重 650、背景使用 `$bg-subtle`。
- 行高 40 px，分隔线 1 px。
- 操作列固定右侧，最多两个文字按钮。

### 12.3 Evidence

- 使用“键: 值”两栏排布，键宽 96 px。
- 值最多展示 3 行，超过折叠。
- 来源引用必须可跳到对应文档或预览。

### 12.4 Markdown 差异

- 新增背景使用 `rgba(18,128,92,.10)`。
- 删除背景使用 `rgba(180,35,24,.10)`。
- 不只依赖红绿文字区分差异。

## 13. 动效与响应式约束

| 场景 | 时长 | 曲线 |
|------|------|------|
| 路由切换 | 120 ms | ease-out |
| Drawer / Modal | 160 ms | ease-out |
| Hover / Focus | 100 ms | linear |
| 列表选中 | 120 ms | ease-out |

**规则**

- 动效必须帮助层级、反馈或扫读，不做装饰性漂浮。
- 首版不使用复杂滚动叙事、3D、弹性回弹。
- ≥1280 px 使用三栏工作台。
- 960-1279 px 折叠 Assistant，保留搜索 + 预览。
- <960 px 转为单列 + 底部标签，搜索、上传、预览优先。

---

## 14. 开发减负规则

1. 页面先按工作流拆分 feature，不先抽象复杂设计系统。
2. 同一 UI 模式出现 3 次后再提取共享组件。
3. 所有请求通过统一 API client，页面不直接拼接复杂参数。
4. 服务端状态交给 TanStack Query，Zustand 只保存 UI 状态。
5. 不为“未来可能有的云端多用户”提前设计权限系统。
6. 不把帮助文档和说明文字塞进主工作区。

---

## 15. 验收标准

✅ **产品感**  
首屏像一个正式研究工作台，而不是 demo 拼接页。

✅ **工作流效率**  
用户无需阅读说明即可完成上传、索引、搜索、预览、编辑。

✅ **视觉一致性**  
不同页面截图并排时，颜色、字体、边界和按钮语言统一。

✅ **可维护性**  
工作台页不允许再次形成超过 600 行的单页巨石组件。

✅ **可访问性**  
核心操作可键盘完成，对比度满足 WCAG AA。

✅ **性能**  
列表滚动保持流畅，路由切换本地环境无明显卡顿。
