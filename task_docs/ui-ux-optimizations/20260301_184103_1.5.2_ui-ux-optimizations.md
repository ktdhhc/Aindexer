# UI/UX 优化方案（可直接上手改）

版本: 1.5.2
时间戳: 20260301_184103
适用文件: `backend/frontend/index.html`

## 目标
- 统一按钮/输入/卡片/表格的视觉规范
- 提升异步操作反馈（loading）
- 优化空态引导，降低新手操作成本

## 实施清单（按优先级）

### P1-01 按钮尺寸规范（已实施）
改动点:
- 在 `:root` 新增变量:
  - `--btn-sm: 6px 12px`
  - `--btn-md: 8px 16px`
  - `--btn-lg: 12px 24px`
- `.btn` 默认改用 `var(--btn-md)`
- `.provider-actions .btn` / `.provider-model-editor .btn` 改用 `var(--btn-sm)`
- 新增 `.btn-lg`，用于关键操作按钮

验收:
- Provider 行内按钮不再忽大忽小
- 上传区“一键生成”按钮视觉层级更明确

### P1-02 按钮 loading 态（已实施）
改动点:
- CSS 新增:
  - `.btn.loading`
  - `.btn.loading::after`（旋转圈）
  - `@keyframes spin`
- JS 新增通用包装器:
  - `withButtonLoading(btnEl, worker)`
- 已接入的异步入口:
  - `backupExportAll`
  - `refreshUploadPanel`
  - `runAllIndexing`
  - `searchDocs`
  - `showAllIndexed`
  - `savePreviewMarkdown`

验收:
- 点击上述按钮后出现 loading 动效，期间不可重复点击

### P1-03 输入控件统一（已实施）
改动点:
- `.input/.select/.textarea` 统一 `line-height: 1.5`
- focus ring 统一为 `0 0 0 2px var(--accent-soft)`
- `.textarea-code` 统一为:
  - `font-size: 14px`
  - `line-height: 1.5`
  - `padding: 8px 12px`
  - `min-height: 420px`
  - 背景改用 `var(--card)`

验收:
- 文本输入、下拉、预览编辑框在视觉高度和焦点态上保持一致

### P1-04 卡片与标题区节奏统一（已实施）
改动点:
- `.section-card` 统一 `padding: 16px`
- `.section-header` 统一:
  - `margin-bottom: 12px`
  - `padding-bottom: 12px`

验收:
- 各模块标题区留白一致，无忽松忽紧现象

### P1-05 空态引导强化（已实施）
改动点:
- 上传空态新增按钮: `选择文件上传`
- 搜索空态新增按钮: `清空筛选并查看全部`

验收:
- 空态下不需要用户猜操作路径，1 次点击可回到主流程

### P2-01 表格操作列稳定性（已实施）
改动点:
- `.table th:last-child, .table td:last-child` 增加 `min-width: 120px`

验收:
- 小屏和窄窗口下操作列不挤压到不可点

## 补充修正（实施中顺手修复）
- 修复 `var(--text)` 未定义问题，统一改为 `var(--ink)`

## 仍建议继续做（下一批）

### NEXT-01 交互可访问性
- 为图标按钮补 `aria-label`
- 模态窗增加焦点陷阱（Tab 锁定）和 Esc 关闭

### NEXT-02 响应式按钮组
- `<=480px` 时将顶部操作按钮改为两行分组，避免全宽堆叠过长

### NEXT-03 异步错误提示一致化
- 抽一个 `showToast(type, text)`，替代分散的状态文本写入

## 快速回归脚本（手工）
1) 点击“导出全部/刷新/一键生成/搜索/全部/保存”确认 loading 生效。
2) 键盘 Tab 依次聚焦搜索框、清空按钮、搜索按钮，检查 focus ring 一致。
3) 空态下点击“选择文件上传”和“清空筛选并查看全部”确认路径可达。
4) 缩窄窗口检查表格操作列仍可点击。
