# 项目上下文追加日志

## 只读与追加规则
- 本文件默认采用**只追加（append-only）**方式维护。
- 更新时仅在文末新增内容。
- 既有内容默认为只读，不应修改、删除或重排。
- 仅在三种情况之一更新本文件：用户明确要求、重大功能更新、上下文过长且即将触发记忆压缩前。
- 单个更新点只写 2-3 句概括。
- **唯一允许改动前文的情况：某个功能被移除。**

## 项目概况（初始快照）
- 项目名称：Literature Indexer（文献索引维护系统，V1 向“新手友好 + 稳定性增强”版本演进）。
- 核心目标：提升新手可用性、界面一致性、索引稳定性与运维安全性。
- 运行形态：本地单机，前端页面 + FastAPI 后端。

## 当前架构总览（初始快照）
- 前端核心：`backend/frontend/index.html`
- 后端入口与配置：`backend/app/main.py`、`backend/app/config.py`
- 路由层：`backend/app/routers/index.py`、`backend/app/routers/providers.py`、`backend/app/routers/fields.py`、`backend/app/routers/export.py`、`backend/app/routers/system.py`
- 服务层：`backend/app/services/extractor.py`、`backend/app/services/provider_client.py`
- 数据与仓储层：`backend/app/repository.py`、`backend/app/db.py`、`backend/app/schemas.py`
- 数据目录：`data/uploads/`、`data/indexes/`、`data/exports/`、`data/logs/`，数据库为 `data/app.db`
- 应用日志：`data/logs/app.log`

## 已实现的索引稳定性改造（初始快照）
- 路由层与服务层均增强了取消感知。
- 中断状态链路包含 `cancel_requested` 与 `cancelled`。
- 前端可展示中断清理进度，并在清理期间阻止过早重跑（`cleanup_pending`）。

## 已实现的备份与恢复能力（初始快照）
- 全量导出接口：`GET /api/export/backup/all`（DB + uploads + indexes + logs）。
- 全量恢复接口：`POST /api/export/backup/restore`（覆盖式恢复）。
- 最近安全增强：
  - 恢复前校验 ZIP 条目路径，拦截非法路径与未知顶层目录。
  - 覆盖恢复前自动生成“恢复前快照”。
  - 恢复响应返回 `pre_restore_backup` 文件名。
  - 前端恢复成功提示展示该快照文件名。

## 运维说明（初始快照）

- 恢复操作仍属于强操作（覆盖式），但已具备恢复前快照兜底。
