# 20250315_布局拖拽与视觉统一

## 背景
当前首页采用“上传区 + 下方左右双栏”结构，已支持不可见分隔条拖拽调整比例。但在实际使用中发现以下体验缺陷：

1. 搜索区条目滚动窗口默认高度与“8 条”目标不符，且会被可用空间压缩
2. 卡片之间间隔过紧，缺乏“呼吸感”
3. 可拖动判定区太窄，不易发现
4. 拖动时无视觉提示
5. 搜索工具栏控件高度/排列不一致，拖动时按钮会被压扁
6. 上传区灰色上传区域不随布局伸缩
7. 整体默认比例偏“上重下轻”
8. 刷新后未完全恢复默认比例

本方案一次性解决以上问题，并统一视觉与交互细节。

## 一、需求再确认

1. 搜索区条目滚动窗口固定显示 **8 条**高度，不受当前可用空间压缩，空状态也保持同高
2. 卡片之间间隔加宽，视觉上更“呼吸感”
3. 可拖动判定区加宽，鼠标更容易命中
4. 两条分隔条中部加“短 bar”视觉提示，让用户一眼发现“这里可以拖”
5. 搜索框、搜索按钮、全部按钮、下拉栏统一高度，统一靠右排列，仅搜索框宽度随布局拖拽变化
6. 上传区灰色上传区域真正随布局伸缩
7. 整体默认比例微调：上传区默认高度下调，下方区域默认更大，左右默认比例再微调
8. 刷新后强制恢复默认值，避免缓存比例错误

## 二、技术实现要点

### A. 搜索区 8 条高度强制落地
- 关键位置：`backend/frontend/index.html:4096` 附近的 `syncPreviewSearchHeights()`
- 删除 `Math.min(ideal, available)`，直接取 `ideal`
- 给 `.search-table-wrap` 和 `#searchEmpty` 都加 `min-height: target`

### B. 卡片间隔加宽
- 把 `--layout-splitter` 从 `10px` 提到 `14px`
- 把 `.main-layout` 和 `.bottom-grid` 的 `gap` 从 `0` 提到 `12px`
- 同步调整 splitter 的 `::before` 占位

### C. 可拖动判定区加宽 + 中部短 bar
- 分隔条 `14px`
- 中部加 `2px * 24px` 短 bar，颜色 `--line` 浅变体，hover 加深

### D. 搜索工具栏统一高度 + 仅搜索框弹性
- 统一 `height: 38px`
- 去掉按钮/下拉上下 padding 影响
- 搜索框容器 `flex: 1 1 auto`，其余 `flex: 0 0 auto`
- 外层 `.search-toolbar` 设为 `flex-wrap: nowrap` + 靠右

### E. 上传区灰色上传区域真正随布局伸缩
- 给 `.upload-section .section-body` 加 `flex: 1`
- `#uploadEmpty` 默认 `min-height: 90px` 并参与伸缩
- 空状态也同高

### F. 整体比例微调 + 刷新强制恢复
- 默认 `--layout-top-size: 280px`
- 默认 `--layout-left-size: 62%`
- 刷新时显式清除内联样式，避免缓存比例

## 三、具体落地步骤

### 1. 统一搜索控件高度与排列
```css
.search-toolbar {
  margin-left: auto;
  justify-content: flex-end;
  align-items: stretch;
  flex-wrap: nowrap;
  width: auto;
  flex: 0 1 auto;
}
.search-toolbar .btn,
.search-toolbar .select,
.search-toolbar .search-input-wrap,
.search-toolbar .input {
  height: 38px;
  box-sizing: border-box;
}
.search-toolbar .btn,
.search-toolbar .select {
  flex: 0 0 auto;
  padding-top: 0;
  padding-bottom: 0;
}
.search-input-wrap {
  flex: 1 1 auto;
  min-width: 0;
}
.search-input-wrap .input {
  width: 100%;
  min-width: 0;
  padding-right: 52px;
}
```

### 2. 强制 8 条高度
```javascript
const rowCount = 8;
const header = searchWrap.querySelector('thead');
const firstRow = searchWrap.querySelector('tbody tr');
const headerH = header ? Math.ceil(header.getBoundingClientRect().height) : 44;
const rowH = firstRow ? Math.ceil(firstRow.getBoundingClientRect().height) : 46;
const target = headerH + rowH * rowCount + 2;
searchWrap.style.flex = '0 0 auto';
searchWrap.style.minHeight = hPx;
searchWrap.style.height = hPx;
searchWrap.style.maxHeight = hPx;
if (searchEmpty) {
  searchEmpty.style.minHeight = hPx;
  searchEmpty.style.height = hPx;
  searchEmpty.style.maxHeight = hPx;
}
```

### 3. 卡片间隔加宽
```css
.main-layout {
  gap: 12px;
}
.bottom-grid {
  gap: 12px;
}
.layout-splitter {
  width: 14px;
}
```

### 4. 中部短 bar 视觉提示
```css
.layout-splitter-horizontal::after {
  content: "";
  position: absolute;
  left: 50%;
  top: 50%;
  transform: translate(-50%, -50%);
  width: 24px;
  height: 2px;
  background: var(--line);
  border-radius: 1px;
}
.layout-splitter:hover::after {
  background: var(--accent);
}
```

### 5. 上传区灰色上传区域真正随布局伸缩
```css
.upload-section .section-body {
  flex: 1;
  min-height: 0;
}
#uploadEmpty {
  flex: 1;
  min-height: 90px;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  gap: 6px;
}
```

### 6. 整体比例微调 + 刷新强制恢复
```javascript
function applyLayoutDefaults() {
  const mainLayout = document.getElementById('mainLayout');
  if (!mainLayout) return;
  mainLayout.style.removeProperty('--layout-top-size');
  mainLayout.style.removeProperty('--layout-left-size');
  mainLayout.style.removeProperty('height');
  mainLayout.style.setProperty('--layout-top-size', '280px');
  mainLayout.style.setProperty('--layout-left-size', '62%');
}
window.addEventListener('pageshow', () => {
  applyLayoutDefaults();
  syncMainLayoutHeight();
  syncPreviewSearchHeights();
});
```

## 四、验证点（实施后可自测）

1. 搜索区滚动窗口是否稳定显示 8 条（无论窗口大小）
2. 拖动左右分隔条时，是否只有搜索框变宽，按钮/下拉不乱动
3. 卡片之间是否明显更宽、更“呼吸”
4. 鼠标移到分隔条中部是否能看到短 bar，hover 会加深
5. 拖动上传区高度时，灰色上传区域是否跟着一起伸缩
6. 刷新后是否恢复新的默认布局
